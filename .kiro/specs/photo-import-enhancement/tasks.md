# 照片导入功能完善实施计划

## 任务概述

本实施计划将照片导入功能完善分解为具体的编码任务，按照优先级和依赖关系组织。每个任务都包含明确的实现目标和验收标准，确保功能的渐进式开发和测试。

**核心导入规则：** 所有文件导入时严格以时间为准进行日期归类，按照 YYYY/YYYYMMDD 的目录结构组织文件。时间获取优先级为：EXIF拍摄时间 > 视频元数据时间 > 文件创建时间。

## 现有实现分析和复用性评估

### 可以复用的现有功能：

1. **基础文件扫描** - `scanFolder()` 函数可以复用，已支持递归扫描和文件类型识别
2. **EXIF日期提取** - `resolveExifDate()` 和 `checkExifDate()` 可以复用和增强
3. **文件复制逻辑** - `copyFile()` 函数的重复处理逻辑可以复用
4. **目录创建** - `ensureDir()` 函数可以直接复用
5. **基础UI框架** - 导入对话框的基本结构可以保留
6. **IPC通信模式** - 现有的 preload API 模式可以扩展

### 需要新增的功能：

1. **Worker架构** - 需要创建新的 import-worker 和 import-service
2. **视频元数据提取** - 全新功能，需要集成 ffprobe
3. **HEIC支持增强** - 需要在 main 进程中实现
4. **文件组检测** - 全新功能
5. **多种重复处理策略** - 需要扩展现有的重命名逻辑
6. **导入历史和撤销** - 全新功能
7. **高级UI组件** - 预览、过滤、进度监控等

### 复用率估算：

- **核心逻辑复用率**: ~40% (文件扫描、EXIF处理、文件操作)
- **UI组件复用率**: ~30% (基础对话框结构、进度显示)
- **架构模式复用率**: ~60% (Worker模式、IPC通信、类型定义)
- **整体复用率**: ~45%
