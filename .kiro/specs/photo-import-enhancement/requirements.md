# 照片导入功能完善需求文档

## 介绍

本文档定义了对Photasa照片导入功能的完善需求。当前的导入功能虽然基本可用，但在用户体验、灵活性和性能方面还有很大改进空间。我们将按照优先级分阶段完善：优先解决导入体验问题，然后增加灵活的导入策略，最后进行性能和高级功能优化。

**核心导入规则：** 所有文件导入时严格以时间为准进行日期归类，按照 YYYY/YYYYMMDD 的目录结构组织文件。时间获取优先级为：EXIF拍摄时间 > 视频元数据时间 > 文件创建时间。

## 需求

### 需求 1：改进导入用户体验

**用户故事：** 作为用户，我希望有一个直观、清晰的导入界面，能够清楚地了解导入进度和状态，以便更好地控制导入过程。

#### 验收标准

1. WHEN 用户打开导入对话框 THEN 界面应该只显示必要的导入选项，移除无关的Activity type和Resources选项
2. WHEN 用户开始导入 THEN 应该显示详细的进度信息，包括当前处理的文件、已完成数量、总数量、处理速度和预估剩余时间
3. WHEN 导入过程中 THEN 用户应该能够取消或暂停导入操作
4. WHEN 导入完成或出错 THEN 应该显示清晰的结果摘要，包括成功导入的文件数、跳过的文件数和错误信息
5. WHEN 导入过程中出现错误 THEN 应该显示具体的错误信息和建议的解决方案

### 需求 2：增强导入前预览功能

**用户故事：** 作为用户，我希望在正式导入前能够预览将要导入的文件，并能够选择性地导入，以便更好地控制导入内容。

#### 验收标准

1. WHEN 用户选择源目录后 THEN 应该自动扫描并显示将要导入的文件列表
2. WHEN 显示文件列表 THEN 应该包含文件缩略图、文件名、大小、修改日期和目标路径预览
3. WHEN 用户查看文件列表 THEN 应该能够选择或取消选择特定文件进行导入
4. WHEN 用户查看文件列表 THEN 应该显示导入统计信息（总文件数、总大小、按类型分组统计）
5. WHEN 检测到重复文件 THEN 应该在列表中标记并提供处理选项预览

### 需求 3：预配置的重复文件处理策略

**用户故事：** 作为用户，我希望能够在导入前选择重复文件处理策略，这样导入过程可以自动处理重复文件而无需中断。

#### 验收标准

1. WHEN 用户配置导入设置 THEN 应该提供重复文件处理策略选择：跳过、重命名、覆盖、保留两者
2. WHEN 用户选择处理策略 THEN 该策略应该应用于导入过程中遇到的所有重复文件
3. WHEN 在预览步骤中 THEN 应该标识出重复文件但不需要用户交互
4. WHEN 导入执行时 THEN 应该根据预设策略自动处理重复文件
5. WHEN 导入完成后 THEN 应该在结果中显示重复文件的处理统计

### 需求 4：多目录和批量导入支持

**用户故事：** 作为用户，我希望能够同时从多个目录导入照片，并支持批量操作，以便提高导入效率。

#### 验收标准

1. WHEN 用户选择源目录 THEN 应该支持选择多个目录进行批量导入
2. WHEN 选择多个源目录 THEN 应该显示所有目录的文件统计汇总
3. WHEN 执行批量导入 THEN 应该按目录分组显示进度信息
4. WHEN 批量导入过程中 THEN 用户应该能够暂停或取消特定目录的导入
5. WHEN 批量导入完成 THEN 应该显示按目录分组的导入结果报告

### 需求 5：文件过滤和筛选功能

**用户故事：** 作为用户，我希望能够根据文件类型、大小、日期等条件过滤要导入的文件，以便只导入需要的文件。

#### 验收标准

1. WHEN 用户查看待导入文件列表 THEN 应该提供文件类型过滤选项（图片、视频、全部）
2. WHEN 用户设置过滤条件 THEN 应该支持按文件大小范围过滤
3. WHEN 用户设置过滤条件 THEN 应该支持按修改日期范围过滤
4. WHEN 应用过滤条件 THEN 文件列表和统计信息应该实时更新
5. WHEN 用户清除过滤条件 THEN 应该恢复显示所有符合条件的文件

### 需求 6：导入历史和恢复功能

**用户故事：** 作为用户，我希望能够查看导入历史记录，并在需要时能够撤销最近的导入操作，以便管理导入的文件。

#### 验收标准

1. WHEN 导入完成 THEN 系统应该记录导入操作的详细信息（时间、源目录、目标目录、文件列表）
2. WHEN 用户查看导入历史 THEN 应该显示最近的导入操作列表
3. WHEN 用户选择历史记录 THEN 应该显示该次导入的详细信息和文件列表
4. WHEN 用户选择撤销导入 THEN 应该能够删除该次导入的所有文件并更新配置
5. WHEN 执行撤销操作 THEN 应该显示确认对话框并提供撤销操作的详细预览

### 需求 7：智能文件组导入

**用户故事：** 作为用户，我希望系统能够识别并整组导入相关联的文件（如MP4视频及其伴随的THM、LRV等文件），以便保持文件的完整性和关联性。

#### 验收标准

1. WHEN 检测到主文件（如MP4视频） THEN 系统应该自动识别同名的关联文件（如.THM、.LRV、.SRT等）
2. WHEN 显示待导入文件列表 THEN 应该将文件组作为一个单元显示，并标明包含的文件类型
3. WHEN 用户选择导入文件组 THEN 应该确保整组文件都被导入到相同的目标目录
4. WHEN 处理文件组的重复检测 THEN 应该基于主文件进行重复判断，但处理时包含整个文件组
5. WHEN 文件组中部分文件缺失或损坏 THEN 应该显示警告但仍允许导入可用的文件

### 需求 8：增强元数据提取支持

**用户故事：** 作为用户，我希望系统能够准确提取各种格式文件的元数据信息（包括HEIC、RAW格式图片和各种视频格式），以便正确地按日期组织文件和显示详细信息。

#### 验收标准

1. WHEN 处理HEIC格式图片 THEN 系统应该能够准确提取EXIF信息，包括拍摄日期、GPS位置、相机信息等
2. WHEN 处理RAW格式图片 THEN 系统应该支持主流RAW格式（CR2、NEF、ARW等）的元数据提取
3. WHEN 处理视频文件 THEN 系统应该能够提取视频元数据，包括创建日期、分辨率、编码信息、GPS信息等
4. WHEN 元数据提取失败 THEN 系统应该回退到文件系统日期，并在日志中记录提取失败的原因
5. WHEN 显示文件信息 THEN 应该清楚标明日期来源（EXIF、视频元数据、文件系统）和置信度

### 需求 9：性能优化和大文件处理

**用户故事：** 作为用户，我希望导入大量文件时系统能够保持响应，并提供良好的性能表现，以便高效完成导入任务。

#### 验收标准

1. WHEN 导入大量文件 THEN 系统应该使用批处理机制避免界面卡顿
2. WHEN 处理大文件 THEN 应该显示单个文件的传输进度
3. WHEN 系统资源紧张 THEN 应该自动调整并发处理数量
4. WHEN 导入过程中 THEN 应该监控内存使用并及时清理临时资源
5. WHEN 导入被中断 THEN 应该能够从中断点恢复导入而不是重新开始
