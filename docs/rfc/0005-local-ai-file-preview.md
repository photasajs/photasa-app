# RFC 0005: 本地AI文件预览功能

## 摘要

本文档描述了一个为Photasa应用添加本地AI文件预览功能的设计方案。该功能将支持Adobe Illustrator (.ai)、Photoshop (.psd)、Sketch (.sketch)、Figma (.figma)、Adobe XD (.xd)等设计文件的本地预览，无需依赖在线服务。

## 背景

当前Photasa应用支持多种图片和视频格式的本地预览，但对于专业设计文件（如AI、PSD等）缺乏预览支持。这些文件格式复杂，需要专业的解析和渲染能力。虽然在线预览服务是长期目标，但本地预览功能能够提供更好的用户体验，特别是在离线环境下。

## 目标

1. 为Photasa应用添加AI文件的本地预览功能
2. 支持主流设计文件格式的预览（优先支持PSD）
3. 与现有的文件预览系统无缝集成
4. 提供良好的性能和用户体验
5. 为未来的在线预览服务奠定基础

### 核心组件

#### 1. 文件类型检测器

- 扩展 `src/main/import/metadata/index.ts` 支持AI文件类型
- 添加AI文件格式识别逻辑
- 集成到现有的文件扫描流程

#### 2. AI文件元数据提取器

- 创建 `src/main/import/metadata/extractors/ai-extractor.ts`
- 实现PSD文件元数据提取（使用ag-psd库）
- 提供基础AI文件信息提取
- 支持错误处理和回退机制

#### 3. 缩略图生成系统

- 扩展 `src/main/thumbnail/thumbnail-handler.ts`
- 添加AI文件缩略图生成逻辑
- 实现PSD文件预览图生成
- 集成到现有的缩略图系统

#### 4. 前端预览组件

- 创建 `src/renderer/src/components/AiFilePreview.vue`
- 扩展 `src/renderer/src/components/MediaPreview.vue`
- 更新 `src/renderer/src/components/import/FilePreview.vue`
- 添加AI文件特有的UI元素

### 技术实现细节

#### 依赖库

```json
{
    "ag-psd": "^1.0.0", // PSD文件解析
    "file-type": "^18.0.0", // 文件类型检测
    "@types/file-type": "^18.0.0" // TypeScript类型定义
}
```

#### 类型定义扩展

```typescript
// 扩展FileMetadata接口
export interface FileMetadata {
    // ... 现有属性
    // AI文件特有属性
    layers?: number;
    colorMode?: string;
    version?: string;
    hasTransparency?: boolean;
    artboardCount?: number;
}

// 添加AI文件类型
export type FileType = "image" | "video" | "ai" | "all";
```

#### 元数据提取流程

1. 文件类型检测（基于扩展名）
2. 选择合适的提取器（PSD使用ag-psd，其他使用基础提取器）
3. 提取文件元数据和预览信息
4. 生成缩略图和预览图
5. 缓存结果并返回

#### 错误处理策略

- 对于不支持的格式，提供基础文件信息
- 解析失败时，使用文件系统信息作为回退
- 提供清晰的错误信息和用户反馈
- 记录详细的错误日志用于调试

## 技术挑战和解决方案

### 挑战1：文件格式复杂性

**问题**：AI文件格式复杂，不同格式有不同的内部结构
**解决方案**：

- 优先支持PSD格式（有成熟的JavaScript库）
- 为不支持的格式提供基础信息
- 逐步添加对其他格式的支持

### 挑战2：性能考虑

**问题**：AI文件通常较大，解析和预览生成可能影响性能
**解决方案**：

- 实现异步处理和进度反馈
- 添加缓存机制避免重复处理
- 优化内存使用和垃圾回收

### 挑战3：错误处理

**问题**：AI文件可能损坏或格式不标准
**解决方案**：

- 实现多层错误处理机制
- 提供用户友好的错误信息
- 记录详细日志用于调试
